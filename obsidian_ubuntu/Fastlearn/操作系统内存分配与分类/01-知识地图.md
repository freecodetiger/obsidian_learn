# 知识地图：操作系统内存分配与分类

## 先修知识

- 进程与线程的地址空间概念
- CPU 访问内存的基本路径：虚拟地址 -> 页表 -> 物理地址
- Cache / TLB / Page Fault 的基本含义
- 用户态与内核态切换
- 链表、位图、树等基础数据结构

## 必会概念

- 内存分类
- 进程视角：代码段、数据段、BSS、堆、栈、共享库映射区
- 系统视角：用户空间、内核空间、页缓存、匿名页、文件页、slab/slub 对象缓存
- 物理视角：DMA/Normal/HighMem（经典 Linux 语境）、NUMA 节点、本地内存与远端内存
- 分配粒度：页级分配、小对象分配、用户态堆分配
- 分配接口：`brk/sbrk`、`mmap`、缺页按需分配
- 回收机制：释放、合并、换页、回收页缓存、OOM
- 碎片：内部碎片、外部碎片
- 连续分配与离散分配
- 伙伴系统、slab/slub、用户态 `malloc`

## 心智模型

- 用户进程“看到”的是连续虚拟地址空间，不是连续物理内存。
- 内存分配不是单层问题，而是三层协作：
- 用户态分配器：`malloc/new` 决定小块怎么切。
- 内核虚拟内存层：VMA、页表、缺页异常决定地址区间怎么映射。
- 内核物理内存层：伙伴系统、slab/slub 决定页框和内核对象怎么给。
- 面试里说“申请了内存”必须说清楚：
- 申请的是虚拟地址范围，还是实际物理页。
- 是匿名内存，还是文件映射。
- 是用户态堆，还是内核对象。
- 是立即分配，还是延迟到首次访问时才真正分配物理页。

## 核心机制与不变量

- 不变量 1：进程隔离依赖地址空间隔离，不能直接访问别的进程页表映射。
- 不变量 2：页表必须保证虚拟页到物理页映射的一致性与权限正确性。
- 不变量 3：缺页处理后，CPU 重试指令应能得到合法映射或触发异常。
- 不变量 4：空闲页管理必须支持拆分与合并，否则外部碎片会恶化。
- 不变量 5：小对象频繁分配不能总走页级路径，否则性能与碎片都差。
- 关键流程：
- `malloc(小对象)`：用户态分配器先从自己的 arena/bin/cache 拿；不够再向内核要。
- `malloc(大对象)`：常见实现会直接走 `mmap`。
- 进程首次访问新地址：若尚无物理页，触发缺页异常，内核建立映射并分配页。
- 内核需要连续 `2^k` 页：伙伴系统找对应阶，不够就向上拆分。
- 内核频繁创建小对象：slab/slub 从页中切对象并缓存，减少伙伴系统压力。

## 边界条件

- 栈向下增长，堆通常向上增长，但现代进程更多通过 `mmap` 扩展匿名区，不只是单纯 `brk`。
- `malloc` 成功不等于立刻拿到物理内存；可能只是保留了虚拟地址范围。
- `free` 后内存不一定马上还给内核；很多场景只是回到用户态分配器缓存。
- `mmap` 文件页和匿名页的回收路径不同。
- 页缓存能提高 IO 性能，但会与应用匿名内存竞争可用内存。
- NUMA 机器上“有空闲内存”不代表本地节点有低延迟内存。
- 连续物理内存难拿，长时间运行后高阶页分配更容易失败。
- OOM 不是“总内存用完”这么简单，也可能是某个 cgroup、zone 或高阶分配失败。

## 权衡与对比

- 连续分配 vs 分页分配
- 连续分配地址转换简单，但扩展困难、外部碎片严重。
- 分页分配便于离散物理页拼接虚拟连续空间，但引入页表开销和 TLB 压力。
- `brk` vs `mmap`
- `brk` 适合传统堆增长；`mmap` 更灵活，便于大块分配、文件映射、独立回收。
- 伙伴系统 vs slab/slub
- 伙伴系统擅长页框管理；slab/slub 擅长固定大小小对象管理。
- 预分配 vs 按需分配
- 预分配减少运行时抖动；按需分配减少浪费，但会带来首次缺页延迟。
- 匿名页 vs 文件页
- 匿名页常对应堆栈等运行态数据；文件页可回写或直接丢弃并重载。

## 面试陷阱

- 把“虚拟内存”误说成“磁盘就是内存扩展”。
- 只会背代码段/数据段/堆/栈，不会解释页表、缺页、页缓存。
- 说不清 `malloc`、`mmap`、缺页异常之间的关系。
- 把内存泄漏、内存碎片、内存占用高混为一谈。
- 只会说伙伴系统“二分分配”，说不出为什么需要合并、为什么会有高阶页失败。
- 只会说 slab “提高效率”，说不出它本质是在页内切对象并缓存已初始化元数据。
- 不区分内部碎片和外部碎片。
- 不会把内存管理与性能问题联系到 TLB miss、page fault、swap、NUMA 远端访问。

## 生产环境关注点

- RSS、VSZ、PSS、page cache、swap in/out、major/minor page fault
- OOM killer 与 cgroup memory limit
- 大页与透明大页对吞吐、延迟、内存浪费的权衡
- NUMA 绑定、内存本地性、跨 socket 访问延迟

- 观测入口：`top`、`htop`、`vmstat`、`free`、`pmap`、`smem`、`/proc/<pid>/smaps`、`slabtop`
