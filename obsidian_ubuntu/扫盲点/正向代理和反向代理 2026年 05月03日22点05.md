正向代理：找个人替我发（隐藏自己）
反向代理：找个人替我收（隐藏自己）
#### 1. 正向代理 (Forward Proxy) = 出口 NAPT (SNAT)


- **场景：** 你们寝室的电脑（内网）要访问百度（外网）。
    
- **谁被隐藏了：** **客户端被隐藏了。** 百度只看到路由器的公网 IP，不知道是你这台具体的电脑发出的请求。
    
- **软件层面的例子：** 你用来科学上网的工具、或者公司内网的出口网关。
    
- **网络层面的术语：** **SNAT (Source NAT，源地址转换)**。修改的是发件人地址。
    

#### 2. 反向代理 (Reverse Proxy) = 端口映射 (DNAT)

- **场景：** 你在自己的电脑（内网）跑了一个 Java Spring Boot 网站，你想让全网的人都能访问它。
    
- **谁被隐藏了：** **服务端被隐藏了。** 外部用户访问你们寝室的公网 IP `203.x.x.x:80`，路由器把这个请求“反向”转发到你电脑的 `192.168.1.10:8080` 上。外网用户根本不知道你的内网结构。
    
- **软件层面的例子：** **Nginx**。用户访问 Nginx，Nginx 再把请求分发给背后的多个 Spring Boot 实例。
    
- **网络层面的术语：** **DNAT (Destination NAT，目的地址转换)**，也就是我们常说的“端口映射”或“虚拟服务器”。
    

**总结：** 路由器帮你上网用的 NAPT 是“网络层的正向代理”；而你在路由器上配置端口映射，让外网访问你，那是“网络层的反向代理”。
