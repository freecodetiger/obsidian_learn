对于同一台设备上的进程间通信，有管道、消息队列、共享内存、信号、socket 等多种方式，对于不同设备上进程间通信，就需要网络通信，而设备是不同的，为了兼容不同的设备，统一的网络协议应运而生。


TCP/IP 网络模型：
从上到下依次为：应用层、传输层、网络层、网络接口层

应用层就是我们开发常处理的，比如 websocket，消息队列都是应用层的协议和组件，还有 HTTP\FTP\DNS\SMTP 等，应用层只管如何把用户的数据包装好，不负责传输数据。

应用层在用户态，后面的所有层次都在内核态。

传输层负责将应用层打包好的数据进行分块等可靠处理。
传输层有TCP/UDP 两个传输协议。
**端口号也是传输层给每个数据包添加的，**负责将数据包传输给正确的进程。
每个进程都会被分配一个临时的端口号。

传输层只负责分装好数据包，但是不负责传递数据包，网络层负责。

网络层的协议是 IP 协议，IP 协议将传输层的报文作为数据部分，再添加 IP 头部组成 IP 报文，如果报文长度大于最大 IP 长度，就会在网络层再次分片，最终得到合适的 IP 报文。
通过 IP 寻址可以计算出两台主机是否在同一子网中，子网掩码可以快速判断判断是否在同一子网，以及如果是同一子网也能快速发现机器对应子网内的主机号。
通过IP 协议的路由功能可以决定下一跳，当数据包到达一个网络节点，需要通过路由算法决定下一步走哪条路径。
因此网络层就是找到正确的主机的过程，寻址就像导航，路由就像操控方向盘。

网络接口层负责在 IP 报文的头部增加 MAC 头部，封装成数据帧。
MAC 用来处理同一局域网内的通信，可以通过 ARP 广播来获得对方的 MAC 地址。网络接口层工作在网卡层次，用 mac 地址标识网卡。

总结：应用层（HTTP），传输层（TCP、UDP、端口号），网络层（IP 协议，路由），网络接口层（数据帧，MAC 地址）

国际组织认可 OSI 七层网络模型
从上到下依次为：应用层、**表示层**、**会话层**、传输层、网络层、**数据链路层、物理层**

实际上日常使用更多的是 TCP/IP 模型，实际上，OSI 七层和 TCP/IP 四层有对应关系：
应用层-》应用层+表示层+会话层
传输层-》传输层
网络层-》网络层
网络接口层-》数据链路层+物理层
